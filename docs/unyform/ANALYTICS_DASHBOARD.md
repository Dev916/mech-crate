## Executive Summary

unyform.ai positions itself as the **central nervous system** for AI-assisted development—the single integration point through which all AI code generation flows. This creates a unique opportunity: **the only place in the enterprise that knows exactly how much code is AI-generated, by whom, and whether it conforms to standards.**

This document defines the analytics and metrics that matter to leadership, with a focus on minimizing friction for individual developers while maximizing visibility for engineering leaders.

## 3. AI vs Human Code Analytics

### 3.1 Why This Is The Main Metric

This metric answers the question every engineering leader is asking:

> **"How much of our code is AI-generated, and is that a good thing?"**

Without unyform, organizations have **no visibility** into this. Code enters the codebase through PRs, but there's no attribution of whether it was human-written, AI-suggested, AI-modified, or AI-generated.

**unyform.ai is the only system that knows.**

### 3.2 Code Origin Classification

| Classification | Definition | Example |
|----------------|------------|---------|
| **Human-Only** | Written entirely by developer, no AI assistance | Manual coding |
| **AI-Assisted** | Human-initiated, AI provided suggestions that were modified | Copilot completion, edited |
| **AI-Generated** | Primarily generated by AI, human approved | Chat response, accepted |
| **AI-Modified** | Human code, AI rewrote for conformance | Style reformatting |

## 4. Conformance Index

### 4.1 What Is Conformance?

**Conformance** measures how well code matches organizational standards—regardless of whether it was human or AI-generated.

unyform.ai provides a unique view: **AI code conformance vs Human code conformance**. This answers:

> "Is AI making our codebase more or less consistent?"

### 4.2 Conformance Score Calculation

```
Conformance Score = (1 - (Violations / Checkpoints)) × 100

Where:
- Violations = Rules that failed
- Checkpoints = Total rules evaluated
```

## 11. Privacy & Data Governance

### 11.1 What We Collect

| Data Type | Purpose | Stored | Retention |
|-----------|---------|--------|-----------|
| Request metadata | Analytics | Yes | 2 years |
| Code content | Policy check | No* | N/A |
| Code hashes | Deduplication | Yes | 2 years |
| User identifiers | Attribution | Yes | Account lifetime |
| Conformance scores | Quality tracking | Yes | 2 years |

*Code content is processed in memory for policy evaluation but not persisted.

### 11.2 Privacy Controls

- **Aggregation level:** Configurable (individual, team, org)
- **Data export:** Full GDPR compliance, user data export
- **Retention controls:** Configurable per organization
- **Anonymization:** Optional anonymized reporting mode
