## Architecture

### Leptos SSR + Hydration

```
┌──────────────────────────────────────────────────────────────┐
│                      Browser Request                          │
└────────────────────────────┬─────────────────────────────────┘
                             │
┌────────────────────────────▼─────────────────────────────────┐
│                    Actix-web Server                           │
│   1. Server-side render Leptos components                     │
│   2. Return HTML + hydration scripts                          │
└────────────────────────────┬─────────────────────────────────┘
                             │
┌────────────────────────────▼─────────────────────────────────┐
│                      Browser                                  │
│   1. Display HTML immediately                                 │
│   2. Load WASM bundle                                         │
│   3. Hydrate: attach event handlers                           │
│   4. App becomes fully interactive                            │
└──────────────────────────────────────────────────────────────┘
```

### Actor Model for Stateful Entities

```rust
// Actors manage stateful entities (users, sessions, etc.)
pub struct UserActor {
    user: User,
    db: PgPool,
}

impl Actor for UserActor {
    type Context = Context<Self>;
}

impl Handler<UpdateProfile> for UserActor {
    type Result = Result<User, AppError>;
    
    fn handle(&mut self, msg: UpdateProfile, _ctx: &mut Self::Context) -> Self::Result {
        self.user.name = msg.name;
        // Persist to database
        Ok(self.user.clone())
    }
}
```

## Server Functions

### Defining Server Functions

```rust
#[server(GetUser, "/api")]
pub async fn get_user(id: i32) -> Result<User, ServerFnError> {
    let pool = use_context::<PgPool>().expect("Pool not found");
    
    sqlx::query_as!(User, "SELECT * FROM users WHERE id = $1", id)
        .fetch_one(&pool)
        .await
        .map_err(|e| ServerFnError::ServerError(e.to_string()))
}
```

### Calling from Components

```rust
#[component]
pub fn UserProfile(id: i32) -> impl IntoView {
    let user = create_resource(
        move || id,
        |id| async move { get_user(id).await }
    );
    
    view! {
        <Suspense fallback=move || view! { <p>"Loading..."</p> }>
            {move || user.get().map(|u| match u {
                Ok(user) => view! { <p>{user.name}</p> },
                Err(e) => view! { <p>"Error: "{e.to_string()}</p> },
            })}
        </Suspense>
    }
}
```

## Performance Tips

1. **Use `#[island]`** for interactive components to reduce WASM size
2. **Lazy load routes** with `<Route lazy=true ... />`
3. **Use `create_memo`** for expensive computations
4. **Server functions** for database operations
5. **Enable compression** in production

```rust
// Island component - only this loads WASM
#[island]
fn Counter() -> impl IntoView {
    let (count, set_count) = create_signal(0);
    view! {
        <button on:click=move |_| set_count.update(|n| *n += 1)>
            {count}
        </button>
    }
}
```

## Troubleshooting

### Build Issues

```bash
# Clean all build artifacts
cargo clean

cargo leptos build
```

### WASM Issues

```bash
# Check wasm32 target
rustup target list --installed | grep wasm32

# Add if missing
rustup target add wasm32-unknown-unknown
```

### Hydration Mismatch

- Ensure server and client render identical HTML
- Check for browser-only APIs in SSR code
- Use `#[cfg(feature = "hydrate")]` for client-only code
