## Architecture

### Job Processing Flow

```
┌──────────────┐      ┌──────────────┐      ┌──────────────┐
│   Producer   │──────│    Redis     │──────│   Worker     │
│  (enqueue)   │ pub  │   Pub/Sub    │ sub  │  (process)   │
└──────────────┘      └──────────────┘      └──────┬───────┘
                                                    │
                                            ┌───────▼───────┐
                                            │  PostgreSQL   │
                                            │  (job state)  │
                                            └───────────────┘
```

### Worker Internals

```
┌─────────────────────────────────────────────────────────────┐
│                      Worker Process                          │
│  ┌─────────────┐   ┌─────────────┐   ┌─────────────┐       │
│  │ Subscriber  │──▶│ Job Queue   │──▶│ Processor   │       │
│  │  (Redis)    │   │ (bounded)   │   │  (async)    │       │
│  └─────────────┘   └─────────────┘   └──────┬──────┘       │
│                                              │              │
│                    ┌─────────────────────────▼────────────┐ │
│                    │         Thread Pool (N workers)      │ │
│                    │  ┌────┐ ┌────┐ ┌────┐ ┌────┐        │ │
│                    │  │ W1 │ │ W2 │ │ W3 │ │ W4 │        │ │
│                    │  └────┘ └────┘ └────┘ └────┘        │ │
│                    └──────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────┘
```

## Job Types

### Defining Jobs

```rust
// src/domain/models/job.rs
#[derive(Debug, Serialize, Deserialize)]
pub struct Job {
    pub id: Uuid,
    pub job_type: JobType,
    pub payload: serde_json::Value,
    pub status: JobStatus,
    pub attempts: i32,
    pub max_attempts: i32,
    pub created_at: DateTime<Utc>,
    pub started_at: Option<DateTime<Utc>>,
    pub completed_at: Option<DateTime<Utc>>,
}

#[derive(Debug, Serialize, Deserialize)]
pub enum JobType {
    Process,
    Notify,
    Cleanup,
    LlmInference,
}

#[derive(Debug, Serialize, Deserialize)]
pub enum JobStatus {
    Pending,
    Processing,
    Completed,
    Failed,
    Retrying,
}
```

### Processing Jobs

```rust
// src/worker/processor.rs
impl Processor {
    pub async fn process(&self, job: Job) -> Result<(), JobError> {
        match job.job_type {
            JobType::Process => self.handle_process(job).await,
            JobType::Notify => self.handle_notify(job).await,
            JobType::Cleanup => self.handle_cleanup(job).await,
            JobType::LlmInference => self.handle_llm(job).await,
        }
    }
}
```

## Graceful Shutdown

The worker handles `SIGTERM` gracefully:

1. Stop accepting new jobs
2. Wait for in-flight jobs to complete
3. Save job state to database
4. Exit cleanly

```bash

docker kill --signal=SIGTERM {{SERVICE_NAME}}

# Or in Kubernetes

```

## Error Handling & Retries

```rust
// Jobs are retried with exponential backoff
impl RetryPolicy {
    pub fn should_retry(&self, job: &Job) -> bool {
        job.attempts < job.max_attempts
    }
    
    pub fn next_delay(&self, attempts: i32) -> Duration {
        // Exponential backoff: 1s, 2s, 4s, 8s, ...
        Duration::from_secs(2_u64.pow(attempts as u32))
    }
}
```
