### Production Build

Production builds are optimized for minimal size and fast startup:

- ✅ Only `.output` directory (no source code)
- ✅ No node_modules in final image
- ✅ Non-root user execution
- ✅ dumb-init for proper signal handling
- ✅ ~50-100MB final image size

```bash

mx build {{SERVICE_NAME}} --prod -t v1.0.0            # With version tag

make build-prod s={{SERVICE_NAME}}                    # Production build (alias)
make build s={{SERVICE_NAME}} prod=1 t=v1.0.0         # With tag
make build s={{SERVICE_NAME}} prod=1 t=v1.0.0 push=1  # Build and push
```

## Rendering Modes

### SSR (Server-Side Rendering) - Default

```typescript
// nuxt.config.ts
export default defineNuxtConfig({
  ssr: true,
})
```

Best for: SEO-critical pages, dynamic content

### SPA (Single Page Application)

```typescript
// nuxt.config.ts
export default defineNuxtConfig({
  ssr: false,
})
```

Best for: Dashboards, authenticated apps

### Static (SSG)

```bash
npm run generate
```

Best for: Marketing sites, blogs, documentation

### Hybrid (Per-Route)

```typescript
// pages/dashboard.vue
definePageMeta({
  ssr: false  // This page renders client-side only
})
```

### Environment Variables

```bash
# Server-only (not exposed to client)

DATABASE_URL=postgres://user:pass@host/db
```

### Runtime Config

```typescript
// nuxt.config.ts
export default defineNuxtConfig({
  runtimeConfig: {
    // Server-only
    apiSecret: process.env.API_SECRET,
    
    // Public (client + server)
    public: {
      apiBase: process.env.NUXT_PUBLIC_API_BASE || '/api',

})
```

Access in components:

## Performance Tips

1. **Use `<NuxtImg>`** for optimized images
2. **Lazy load components** with `<LazyComponent />`
3. **Use `useFetch` with `key`** for proper caching
4. **Enable compression** in production
5. **Use `useAsyncData`** for server-side data fetching

```typescript
// Lazy loading
const LazyHeavyComponent = defineAsyncComponent(() => 
  import('~/components/HeavyComponent.vue')
)

// Data fetching with cache
const { data } = await useFetch('/api/posts', { key: 'posts' })
```
